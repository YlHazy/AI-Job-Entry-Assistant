# Skill: development_workflow

## 目的
为“岗位录入助手”项目提供统一的日常开发流程，确保每次需求迭代都按同一套节奏推进。

## 使用时机
- 开始实现任何新功能前
- 修复 bug 或重构前
- 准备提交代码前

## 开发流程
1. 先阅读 `README.md`、`README_dev.md` 和本目录下相关业务 skill，确认当前版本目标仍然是 V1 主链路优先。
2. 开工前先明确本次改动属于哪一类：解析、分类、Excel 写入、UI、测试、文档。
3. 优先保证主流程可运行，不要在 V1 阶段提前扩展复杂功能。
4. 每次实现都要同步更新相应的测试或验证脚本，不能只改业务代码。
5. 完成实现后先自测，再做代码审查清单检查，最后再准备 git 提交。

## 实施原则
- 小步提交，避免一次性混入不相关改动。
- 优先修主链路阻塞问题，再做体验优化。
- 遇到不确定规则时，宁可返回空值，也不要强行猜测。
- 新增代码要与现有模块职责保持一致，避免把逻辑堆进 `app.py`。
