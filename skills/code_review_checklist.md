# Skill: code_review_checklist

## 目的
为本项目提供提交前自审清单，降低“代码能跑但行为不稳”的风险。

## 使用时机
- 每次完成开发后、提交前必须过一遍

## 代码审查清单
1. 这次改动是否只解决一个明确问题，还是混入了无关改动？
2. 模块职责是否清晰？解析逻辑、分类逻辑、Excel 写入逻辑是否仍在各自文件中？
3. 是否存在明显硬编码路径、列名、规则散落多处的问题？
4. 对无法判断的信息，是否安全返回空值或默认值，而不是编造结果？
5. 报错信息是否能帮助定位问题，而不是简单抛出模糊异常？
6. 是否补了对应测试，测试名称是否能说明覆盖了什么行为？
7. README 或开发文档是否需要同步更新？

## 重点关注
- Excel 写入是否会误写错 sheet 或错列
- 分类关键词是否有相互冲突导致误判
- 文本解析是否过度依赖单一格式
- UI 是否把底层异常直接吞掉
