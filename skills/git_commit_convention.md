# Skill: git_commit_convention

## 目的
规范本项目的 git 工作方式，确保每次提交都可读、可追踪、可回滚。

## 使用时机
- 每次准备提交代码时必须执行

## 提交前要求
1. 先确认改动范围聚焦，只包含当前任务直接相关文件。
2. 提交前必须完成本地测试或最少的针对性验证，不能跳过验证直接提交。
3. 提交说明必须写清楚“做了什么”，不要只写“update”“fix”“修改一下”。
4. 如果改动影响行为、字段映射或规则判断，提交说明里要体现影响点。

## 推荐提交格式
使用简洁中文或中英混合均可，但要包含动作和对象，例如：
- `feat: 实现 JD 文本解析与分类主链路`
- `fix: 修正 Excel JobBoard 列映射逻辑`
- `test: 补充 parser 和 classifier 单测`
- `docs: 更新本地运行说明与开发约定`

## 禁止事项
- 不要把无关格式化、临时调试代码、注释垃圾一起提交。
- 不要在未查看 diff 的情况下直接提交。
- 不要跳过测试结果说明。
